CODE DESIGN

We built the code in a modular design, splitting the problem into smaller tasks and writing functions for each task. We take the input as a string array storing each line at the corresponding index.. We validate the proof line by line. We first split the line into the statement, rule, line1 and line2 storing them in the respective arrays. We identify the rule in the line and make the respective function call based on the rule. The statement is then validated based on the rule and the respective index in the ans[] array is made either 0 or 1 based on whether the statement is valid or not...

ALGORITHM

overall ALGORITHM

We use a for loop to take each line of proof as the string at each index of proof[] array. We pass the line using the split_line() function to split the proof line with proper indexing into the string arrays of statement[] and rule[] and integer arrays line1[] and line2[] to store the respective proof lines of the statement. We then traverse through the rule array from index 1 to index = n(no. of lines of proof). We pass the rule through to to rule_check() function and verify the respective rule with the respective function call. While verifying we edit values of the ans[] array. Finallly a for loop is used to traverse through the ans array to check if the values within the range inluding 1 and no.of lines is equal to 1 which implies the validity of that line. If all lines have validity = 1 then we conclude the proof is valid or else we conclude the proof is invalid.
